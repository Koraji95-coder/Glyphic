Binary Search Trees

A BST is a binary tree where each node's left subtree contains only nodes with keys less than the node's key, and the right subtree contains only keys greater than the node's key.

Key Operations

  • search(key) — O(log n) average, O(n) worst
  • insert(key) — O(log n) average
  • delete(key) — O(log n) average, requires handling 3 cases
Exam tip: Always mention the worst-case for unbalanced trees degenerating into linked lists.

AVL Trees

Self-balancing BST where the balance factor (height difference of left and right subtrees) of any node is at most 1. Uses rotations to maintain balance after insertions and deletions.

function rotateRight(node):
    newRoot = node.left
    node.left = newRoot.right
    newRoot.right = node
    updateHeight(node)
    updateHeight(newRoot)
    return newRoot

Hash Tables

Hash tables provide O(1) average-case lookup by mapping keys to array indices via a hash function. Collision resolution strategies include:

  • Chaining — linked lists at each bucket
  • Open Addressing — linear probing, quadratic probing, double hashing
